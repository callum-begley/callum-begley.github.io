What does it mean to display inline vs inline blocks?

4 April 2025

Display: Block vs. Inline vs. Inline-Block

Display is the property that specifies how an element is displayed.

All elements have a default display property. But these can be manipulated in CSS.

Block elements take up the maximum space, or the whole line, and have a width and height, eg. h1, p, div.

Inline elements take up the minimum space possible, and appear on the same line, but do not have a width and height, eg. a, span.

Inline-block elements also take up the minimum space possible, but have a width and height, eg. img



I have set up a simple page to show how these displays work:

HTML CSS Output

As you can see there are two div and two span elements.

The div elements (block) take up a whole line each and the span's (inline) take up the minimum space and appear on the same line.

Now I have added a height of 100px to each. The block elements have increased in height, but the inline ones have not changed.

I have now added an img element (inline-block) between the two div's. Since the div's are block elements the img is on it's own line.

Now I have put the img between the two span elements. You can see that it acts like an inline element, taking up the minimum space and appearing on the same line.

Now the img is between the div and span elements. I have changed the display of the elements in CSS, without changing their HTML. (Please note the text in the image above is no longer correct.)

The blue blocks have been changed to display: inline; They share the same line but no longer have a height of 100px.

The img is now set to display: block; It takes up a whole line and does not appear by the inline elements above.

The red blocks have been set to display: inline-block; They appear on the same line but now can take on the height of 100px.

Lastly, I have set the img to display: none; This makes the image disappear and the other elements act as if it's not there.
